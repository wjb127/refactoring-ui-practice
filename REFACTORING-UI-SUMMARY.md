# Refactoring UI - 핵심 내용 정리

## 1. Starting from Scratch (처음 시작하기)

### 레이아웃이 아닌 기능부터 시작하라
- 네비게이션, 사이드바 같은 "껍데기"부터 디자인하지 말 것
- 실제 기능(예: 항공편 검색 폼)부터 시작하라
- Google처럼 핵심 기능만으로도 충분할 수 있다

### 디테일은 나중에
- 초기 단계에서 폰트, 그림자, 아이콘에 집착하지 말 것
- **그레이스케일로 먼저 디자인** - 색상 없이 spacing, contrast, size로 계층 구조 만들기
- 낮은 피델리티에서 빠르게 움직여라

### 너무 많이 디자인하지 말 것
- 모든 기능을 미리 디자인하려 하지 말 것
- **짧은 사이클로 작업** - 간단한 버전 → 구현 → 반복
- MVP(최소 기능 제품)부터 만들어라

### 개성 선택하기
- **폰트**: Serif = 우아함, Rounded Sans = 친근함, Neutral Sans = 중립적
- **색상**: Blue = 안전함, Gold = 고급스러움, Pink = 재미
- **Border Radius**: 작음 = 중립, 큼 = 친근함, 없음 = 격식있음
- **언어**: 격식체 vs 친근한 말투

### 선택지를 제한하라
- 무한한 옵션은 결정 마비를 유발
- **시스템을 미리 정의**: 색상 8-10개, 폰트 크기 스케일, 여백 시스템
- 매번 새로 결정하지 말고 시스템에서 선택

## 2. Hierarchy is Everything (계층 구조가 전부다)

### 모든 요소가 동등하지 않다
- 시각적 계층 = 요소들의 상대적 중요도
- 모든 것이 주목을 받으려 하면 혼란스럽다
- **의도적으로 덜 중요한 것을 약화**시켜라

### 크기만이 전부가 아니다
- 폰트 크기에만 의존하면 → 중요한 것은 너무 크고, 덜 중요한 것은 너무 작아짐
- **폰트 굵기와 색상**으로 계층 표현
- 3가지 색상: 진한 색(주요), 회색(보조), 연한 회색(3차)
- 2가지 굵기: 보통(400-500), 굵게(600-700)

### 색상 배경에서 회색 텍스트 사용 금지
- 흰색 배경에서 회색 = OK
- 색상 배경에서 회색 = 탁해 보임
- **해결**: 배경색과 같은 Hue로 채도/명도만 조절

### 강조하려면 약화시켜라
- 강조하고 싶은 요소에 효과를 더하는 대신
- **경쟁하는 요소들을 약화**시켜라

### 라벨은 최후의 수단
- "Email: jane@example.com" 대신 그냥 "jane@example.com"
- 포맷과 컨텍스트가 충분히 설명할 수 있다
- "In stock: 12" → "12 left in stock"

### 시각적 계층 ≠ 문서 계층
- h1이라고 항상 커야 하는 건 아니다
- 섹션 제목은 종종 **라벨처럼 작아야** 한다

### 무게와 대비의 균형
- 아이콘(무거움)과 텍스트를 함께 쓸 때 → 아이콘 색상을 연하게
- 얇은 테두리가 너무 약하면 → 두께를 늘려라

### 시맨틱보다 계층이 우선
- Primary: 눈에 띄게 (solid, 고대비)
- Secondary: 명확하지만 돋보이지 않게 (outline, 저대비)
- Tertiary: 발견 가능하지만 눈에 안띄게 (link 스타일)

## 3. Layout and Spacing (레이아웃과 여백)

### 여백은 빼는 것, 더하는 게 아니다
- 처음에 **너무 많은 여백**으로 시작
- 필요한 만큼 줄여나가라

### 간격/크기 시스템 만들기
- 선형 스케일(4px 단위)은 부족
- **기하급수적 스케일** 사용: 4, 8, 12, 16, 24, 32, 48, 64, 96...
- 16px을 기준으로 배수/약수

### 화면 전체를 채울 필요 없다
- 600px이 필요하면 600px만 사용
- 불필요하게 넓히면 가독성 저하

### 그리드에 집착하지 말라
- 12컬럼 그리드는 도구일 뿐
- 사이드바 같은 건 **고정 너비**가 적합
- max-width로 제한하고 필요할 때만 축소

### 상대적 크기는 스케일하지 않는다
- em 단위로 모든 걸 비례시키지 말 것
- 모바일에서 헤드라인 2.5em은 너무 크다
- **독립적으로 조정**하라

### 모호한 간격 피하기
- 라벨과 인풋 사이 < 그룹 사이 간격
- 요소 그룹 내부 간격 < 그룹 외부 간격

## 4. Designing Text (텍스트 디자인)

### 타입 스케일 정하기
- 모듈러 스케일(황금비 등)은 실용적이지 않음
- **직접 선택**: 12, 14, 16, 18, 20, 24, 30, 36, 48, 60, 72px
- em 단위 피하기 (중첩 문제)

### 좋은 폰트 사용하기
- 안전한 선택: 시스템 폰트 스택
- **5개 이상 굵기**가 있는 폰트 선택
- x-height가 높은 폰트 = 가독성 좋음
- 인기 있는 폰트 = 대체로 좋은 폰트

### 줄 길이 체크
- 최적: **45-75자/줄**
- 20-35em 너비 권장

### 베이스라인 정렬
- 다른 크기 텍스트 = 중앙이 아닌 **베이스라인** 정렬

### Line-height는 비례
- 작은 텍스트 → 높은 line-height (1.5+)
- 큰 헤드라인 → 낮은 line-height (1~1.25)
- 넓은 컨텐츠 → 더 높은 line-height

### 모든 링크에 색상 필요 없음
- 본문 속 링크 → 색상으로 구분
- UI의 링크들 → 굵기나 어두운 색상으로

### Letter-spacing 활용
- 헤드라인용 폰트가 아니면 → 좁게
- **대문자 텍스트 → 넓게** (가독성)

## 5. Working with Color (색상 다루기)

### Hex 대신 HSL
- **H**ue (색상, 0-360°)
- **S**aturation (채도, 0-100%)
- **L**ightness (명도, 0-100%)
- 직관적으로 색상 조절 가능

### 생각보다 많은 색상이 필요
- 5가지 색상으로는 부족
- **회색**: 8-10단계
- **주요 색상**: 5-10단계
- **액센트 색상**: 각 5-10단계 (빨강, 노랑, 초록 등)

### 쉐이드를 미리 정의
- lighten/darken 함수 사용 금지
- **100~900 9단계** 미리 정의
- 기본색(500) → 어두운 끝(900) → 밝은 끝(100) → 중간 채우기

### 명도가 채도를 죽인다
- 50% 명도에서 멀어질수록 채도 증가 필요
- **Hue 회전**으로 밝기 조절 (더 생동감)
  - 밝게: 60°, 180°, 300° 방향으로
  - 어둡게: 0°, 120°, 240° 방향으로

### 회색도 회색일 필요 없다
- 차가운 느낌: 파란색 가미
- 따뜻한 느낌: 노란/주황 가미

### 접근성 ≠ 못생김
- WCAG: 일반 텍스트 4.5:1, 큰 텍스트 3:1
- 어두운 배경 + 흰 텍스트가 안되면 → **반전** (밝은 배경 + 어두운 텍스트)

### 색상에만 의존하지 말 것
- 색맹 사용자 고려
- **아이콘, 패턴, 대비**와 함께 사용

## 6. Creating Depth (깊이 만들기)

### 광원 모방하기
- 빛은 위에서 온다
- **위쪽 엣지 = 밝게**, 아래쪽 엣지 = 어둡게
- 돌출 요소: 위에 밝은 테두리, 아래에 그림자
- 오목 요소: 위에 어두운 그림자, 아래에 밝은 테두리

### 그림자로 높이 표현
- 작은 그림자 = 약간 떠있음 (버튼)
- 중간 그림자 = 더 떠있음 (드롭다운)
- 큰 그림자 = 많이 떠있음 (모달)
- **5단계 그림자 시스템** 정의

### 그림자는 두 개로
1. 큰 그림자 (직접광 시뮬레이션) - 넓고 흐릿
2. 작은 그림자 (ambient light 차단) - 좁고 선명

### 플랫 디자인도 깊이 가능
- 밝은 색 = 앞으로, 어두운 색 = 뒤로
- **Solid shadow** (blur 없는 그림자)

### 요소 겹치기
- 카드를 배경 경계에 걸치게
- 이미지 겹칠 때 → "보이지 않는 테두리" 사용

## 7. Working with Images (이미지 다루기)

### 좋은 사진 사용
- 스마트폰 사진으로 대체하려 하지 말 것
- 전문 사진작가 or 고품질 스톡 (Unsplash 등)

### 텍스트는 일관된 대비 필요
- 배경 이미지 위 텍스트가 안보이면:
  - **반투명 오버레이** 추가
  - 이미지 **대비 낮추기**
  - 이미지 **단색화**
  - 텍스트에 **glow 효과** (blur 큰 그림자)

### 의도된 크기가 있다
- 16px 아이콘을 64px로 키우지 말 것 → 뭉툭해 보임
- 해결: 배경 원/사각형 안에 원래 크기로
- 스크린샷 70% 축소 → 텍스트 못 읽음
- 해결: 부분 캡처 or 단순화된 UI 그리기

### 사용자 업로드 컨텐츠 주의
- 고정 컨테이너 + object-fit: cover
- 배경색 비슷하면 → **inner box-shadow**로 구분

## 8. Finishing Touches (마무리)

### 기본 요소 강화
- 불릿 → 체크/화살표 아이콘
- 인용부호 → 크고 다른 색상으로
- 링크 → 두꺼운 커스텀 밑줄
- 체크박스/라디오 → 브랜드 색상

### 액센트 보더로 색상 추가
- 카드 상단에 컬러 라인
- 활성 네비게이션에 컬러 라인
- 알림 메시지 왼쪽에 컬러 라인
- 헤드라인 아래 짧은 컬러 라인

### 배경 꾸미기
- 배경색 변경/그라데이션
- 미묘한 반복 패턴 (Hero Patterns)
- 기하학적 도형/일러스트

### Empty State 신경쓰기
- 빈 화면에 "데이터 없음"만 표시하지 말 것
- 일러스트 + CTA 버튼으로 다음 행동 유도

### 테두리 줄이기
- 구분에 테두리만 사용하지 말 것
- **대안**: 배경색, 그림자, 여백

---

## 핵심 원칙 요약

1. **시스템 먼저** - 색상, 간격, 폰트 크기를 미리 정의
2. **계층 구조** - 모든 요소에 명확한 중요도 부여
3. **제약 = 자유** - 선택지가 적을수록 빠른 결정
4. **디테일은 나중에** - 기능부터, 스타일은 그 다음
5. **독립적 스케일** - 비례가 아닌 맥락에 맞게 조정
6. **빼기의 미학** - 추가보다 제거로 개선
